# Linux CHMOD Permission
หลักการทำงานและที่มาของตัวเลขสิทธิ์

### 1. พื้นฐานสิทธิ์ (Basic Rights)
ใน Linux ทุกไฟล์/ไดเรกทอรีมีสิทธิ์ 3 ด้าน:
|	Short	| Name	|	Description	|
|:---:|:---|:---|
|	r	|	Read	|	อ่านได้	|
|	w	|	Write	|	เขียนได้/แก้ไขได้	|
|	x	|	Execute	|	รันได้/ถ้าเป็นไดเรกทอรีคือเข้าสู่ภายในได้	|

### 2. ผู้รับสิทธิ์ (Permission Target)
สิทธิ์จะถูกแบ่งให้กับ 3 กลุ่ม:
| Name	|	Description	|
|:---|:---|
|	Owner	|	เจ้าของไฟล์ (User)	|
|	Group	|	กลุ่มเจ้าของ (Group)	|
|	Other	|	ผู้ใช้อื่นๆ (World)	|


### 3. ที่มาของตัวเลข (Symbolic -> Binary -> Octal)
ระบบ Linux ใช้ตัวเลขฐาน 8 (Octal) แทนสิทธิ์ ซึ่งมีที่มาจากเลขฐาน 2 (Binary):
```
     ค่าสิทธิ์:   [ r ]   [ w ]   [ x ]
     ---------------------------------------
     (Binary):    2²      2¹      2⁰
                  =       =       =
                  4       2       1
     ---------------------------------------
     (สรุปค่า):   4   :   2   :   1
     ---------------------------------------
```
### 4. การคำนวณสิทธิ์ (Summing for Mode)

สิทธิ์ของแต่ละกลุ่มเกิดจากการเอาค่า (4, 2, 1) มาบวกกัน:
```
     [ r ] + [ w ] + [ x ] =   4 + 2 + 1  = 7   (สิทธิ์ครบถ้วน)
     [ r ] + [ w ] + [ - ] =   4 + 2 + 0  = 6   (อ่าน+เขียน เท่านั้น)
     [ r ] + [ - ] + [ x ] =   4 + 0 + 1  = 5   (อ่าน+รัน เท่านั้น)
     [ r ] + [ - ] + [ - ] =   4 + 0 + 0  = 4   (อ่านอย่างเดียว)
     [ - ] + [ - ] + [ - ] =   0 + 0 + 0  = 0   (ไม่มีสิทธิ์ใดๆ)
```

### 5. การทำงานของคำสั่ง chmod (CHange MODe)
คำสั่ง `chmod` จะกำหนดตัวเลข 3 หลักเพื่อตั้งค่าให้ครบ 3 กลุ่มพร้อมกัน:
```
     ตัวอย่างคำสั่ง:  `chmod 754 [ชื่อไฟล์]`

              \________________
               |     |     |  \
     หลักที่:   1     2     3   [ไฟล์/ไดเรกทอรี]
               |     |     |
               v     v     v
     กลุ่ม:   Owner Group Other
               |     |     |
               |     |     |
     สิทธิ์:   7     5     4
     --------------------------
             [4+2+1] [4+0+1] [4+0+0]
               |     |     |
               v     v     v
             r w x   r - x   r - -
     --------------------------
     ผลลัพธ์:  เจ้าของทำได้ทุกอย่าง,
              กลุ่มอ่าน/รันได้,
              ผู้อื่นอ่านได้เท่านั้น
```

### 6. ตารางสรุปค่าสิทธิ์ยอดนิยม (Common Mode Summary)
|	ตัวเลข	| แสดงสิทธิ์	|	ความเสี่ยง	|	ความหมาย	|
|:---:|:---|:---|:---|
|	777	|	rwxrwxrwx	|	เสี่ยงที่สุด	|	 ทุกคนทำได้ทุกอย่าง |
|	755	|	rwxr-xr-x	|	มาตรฐานโปรแกรม	|	 เจ้าของแก้ไขได้ ผู้อื่นอ่านและรันได้
|	644	|	rw-r--r--	|	มาตรฐานไฟล์ทั่วไป	|	 เจ้าของแก้ไขได้ ผู้อื่นอ่านได้อย่างเดียว
|	600	|	rw-------	|	ไฟล์ลับ	|	 เฉพาะเจ้าของเท่านั้นที่อ่าน/แก้ไขได้
|	700	|	rwx------	|	โฟลเดอร์ลับ	|	 เฉพาะเจ้าของเท่านั้นที่เข้าได้และแก้ไขได้
|	400	|	r--------	|	อ่านอย่างเดียว	|	 เจ้าของอ่านได้อย่างเดียว เช่น ไฟล์ Key	|
