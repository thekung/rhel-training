# Context
ในการทำงานของ SElinux นั้น อย่างที่เราได้ทราบมาแล้วว่า SElinux จะทำงานร่วมกับ Policy ผ่านทาง Context Label ของ object แต่ละตัว

ดังนั้นเราจึงต้องทำความเข้าใจเพิ่มด้วยว่า วิธีการจัดการ Security ของ SElinux นั้นจะอ้างอิงกับ type และ policy เป็นหลัก โดยหากกำหนดค่าต่างๆ ถูกต้องตาม policy และระบบรู้จัก type ที่เกี่ยวข้องกับ policy นั้น SElinux จึงจะอนุญาตให้ process นั้นทำงานได้

สามารถวาด Diagram ความสัมพันธ์ได้ดังนี้
```bash
Process (context: type)
        +
File (context: type)
        ↓
   POLICY (rules)
        ↓
  SELinux DECISION
        ↓
	Allow / Deny
 ```

## วิธีการใช้งาน semanage fcontext
### การเพิ่มกฎใหม่ (Mapping)
สมมติว่าคุณต้องการให้ไฟล์ทั้งหมดใน /custom_web/ มี Context สำหรับ Web Server:
```bash
# 1. เพิ่มกฎเข้าไปในฐานข้อมูล SELinux Policy
sudo semanage fcontext -a -t httpd_sys_content_t "/custom_web(/.*)?"

# 2. ใช้คำสั่ง restorecon เพื่อนำกฎที่ตั้งไว้มาใช้กับไฟล์จริงในปัจจุบัน
sudo restorecon -Rv /custom_web
```

คำอธิบาย:

-a (Add): เพิ่มกฎใหม่

-t (Type): ระบุ Type ที่ต้องการ (เช่น httpd_sys_content_t)

"/custom_web(/.*)?": เป็น Regular Expression หมายถึงตัวโฟลเดอร์และไฟล์/โฟลเดอร์ย่อยทั้งหมดข้างในนั้น

### การตรวจสอบกฎที่มีอยู่
หากต้องการดูว่าระบบมีการตั้งค่า Path ไหนไว้อย่างไรบ้าง
```bash
sudo semanage fcontext -l | grep "/var/www"
```

### การลบกฎ
หากไม่ต้องการกฎนั้นแล้วสามารถสั่งลบได้จากคำสั่ง
```bash
sudo semanage fcontext -d "/custom_web(/.*)?"
```

### สรุปขั้นตอนการทำงาน
**Define:** ใช้ `semanage fcontext` เพื่อบอกระบบว่า "พื้นที่ตรงนี้ควรเป็น Type อะไร"

**Apply:** ใช้ `restorecon` เพื่อเปลี่ยน Label ของไฟล์จริงๆ ให้ตรงตามที่ Define ไว้ในข้อแรก
