# 6.1 SElinux

SELinux (Security-Enhanced Linux) คือ ระบบควบคุมความปลอดภัยระดับลึกของ Linux เพื่อเพิ่มความปลอดภัยมากกว่า permission ปกติอย่าง `chmod`, `chown`

SElinux จะควบคุมว่า "โปรเซสอะไร ทำอะไรกับไฟล์ไหนได้บ้าง" เรียกสิ่งนี้ว่า `MAC` (Mandatory Access Control)

ระบบความปลอดภัยของ Linux ก่อนที่จะมี SElinux นั้นจะใช้เป็นแบบ `DAC` (Discretionary Access Control) ซึ่งทั้ง DAC และ MAC แตกต่างกันคือ

|	ชนิด	|	คำอธิบาย	|
|:---|:---|
|	DAC	|	กำหนดสิทธิ์โดยเจ้าของไฟล์	|
|	MAC	|	กำหนดสิทธิ์โดยระบบ แม้แต่ root ก็ผ่าฝืนกฎไม่ได้	|

### คำอธิบายเพิ่มเติม:
|	Name	|	Description	|
|:-----|:-----|
| **SELinux** | เป็น Linux Kernel Module ที่ช่วยเสริมสร้างความปลอดภัยให้กับระบบปฏิบัติการ Red Hat Enterprise Linux (RHEL) โดยการตรวจสอบการเข้าถึงทรัพยากรต่างๆ ของระบบตามนโยบาย (Policy) ที่กำหนดไว้ |
| **Policy** | เป็นชุดของกฎ (Rules) ที่กำหนดว่ากระบวนการ (process) ใดสามารถเข้าถึงวัตถุ (Object) ใดได้บ้าง โดยการตรวจสอบความสอดคล้องกันของ Context ของกระบวนการและวัตถุนั้นๆ ซึ่ง Policy จะประกอบด้วยกฎ (Rules) และการจับคู่ (Matches) ระหว่าง Type, Role, and User ของ Context	|
| **Context** | เป็นป้ายกำกับ (Label) ที่ใช้ระบุประเภทของวัตถุ (Object) ในระบบ เช่น ไฟล์ (file), กระบวนการ (process), พอร์ต (port), เป็นต้น โดยแต่ละ Context จะประกอบด้วยข้อมูลสำคัญ 3 ส่วนคือ: |

### ส่วนประกอบของ Context
|	Name	|	Description	|
|:-----|:-----|
| **Type** | ระบุประเภทของวัตถุ (เช่น httpd_t คือเว็บเซิร์ฟเวอร์ Apache, file_t คือไฟล์ทั่วไป)	|
| **Role** | ระบุบทบาทของวัตถุ (เช่น system_r คือบทบาทของระบบ, user_r คือบทบาทของผู้ใช้)	|
| **User** | ระบุผู้ใช้ที่เกี่ยวข้องกับวัตถุ (เช่น httpd_u คือผู้ใช้ Apache, user_u คือผู้ใช้ทั่วไป)	|

## SELinux ทำงานอย่างไร
หลักการทำงานของ SElinux จะประกอบไปด้วยสองส่วนสำคัญคือ
- Context
- Policy

โดยที่ทุกอย่างในระบบจะต้องมีการกำหนด `Context` เสมอเพื่ออ้างอิงการบังคับใช้งาน `Policy` ของระบบให้เข้ากับ `Context` นั้นๆ โดยระบุผ่านทาง `Label` ดังนั้นทุก object ของระบบจะต้องระบุชื่อหรือ `Label` เพื่อไว้อ้างอิงกับ `Context` ที่จะใช้งานกับ `Policy` ต่างๆ ของระบบจึงจะผ่านตามกฏของ SElinux และสามารถทำงานได้

## รูปแบบของ Context ใน SElinux
การตัดสินอนุญาตหรือไม่สำหรับ SElinux นั้นจะมุ่งไปที่ type ของ context โดยใน full context จะประกอบไปด้วยข้อความเต็มลักษณะนี้
```Bash
user:role:type:level
```
|	ชนิด	|	คำอธิบาย	|
|:---|:---|
|	user	|	context นี้เป็นของใคร ผู้ใช้ไหน (ไม่จำเป็นต้องเป็นผู้ใช้ที่สร้างขึ้นสำหรับเข้าใช้งาน Linux)	|
|	role	|	ส่วนที่กำหนดว่าสิ่งนี้คืออะไร	|
|	type	|	ชื่อที่ใช้ผูกกับ policy เพื่อให้ SElinux อนุญาตหรือบล็อคการทำงาน	|
|	level	|	ระดับขั้นความสำคัญ	|

** SElinux ตัดสินการอนุญาตของ oject นั้นๆ ด้วย `type` เป็นหลัก

## Mode ของ SElinux
SElinux ที่ทำงานใน RHEL โดยปกติจะเป็นค่าบังคับใช้คือ `Enforcing` เพื่อมาตรฐานความปลอดภัยสูงสุด

วิธีตรวจสอบการทำงานของ SElinux ด้วยคำสั่ง `getenforce` ตัวอย่างเช่น
```bash
root@rhel01:~# getenforce
Enforcing
root@rhel01:~#
```
ซึ่งสถานะที่แสดงออกมาคือสถานะปัจจุบันว่า SElinux ถูกเปิดใช้งานอยู่หรือไม่ ซึ่งจะเป็นไปได้ 3 สถานะด้วยกันดังนี้
|	สถานะ/Config	|	ความหมาย	|
|:--------|:----------|
|	Enforceing	|	SElinux เปิดทำงานอยู่ (Default)	|
|	Permissive	|	SElinux ปิดทำงานชั่วคราว	|
|	Disabled	|	SELinux ไม่ถูกเปิดใช้งาน	|

## คำสั่งที่เกี่ยวข้อง
ในการตรวจสอบ `Context` ของไฟล์ สามารถใช้คำสั่ง `ls` และระบุพารามิเตอร์ `-Z` ดังนี้
```bash
ls -Z
```
ตัวอย่างรายละเอียด context ของไฟล์/Directory เช่น
```bash
system_u:object_r:system_conf_t:s0 yum.repos.d
```
จากตัวอย่างจะเห็นว่า `yum.repos.d` นั้นมีเจ้าของ Context เป็น `system_u` และมี Label เป็น `system_conf_t` อธิบายได้ตามตารางนี้
|	ชนิด	|	คำอธิบาย	|
|:---|:---|
|	system_u	|	เป็นชื่อผู้ใช้ของระบบใน Linux เอง ซึ่งเป็น system user ไม่ใช่ user ที่เข้าใช้งาน	|
|	object_r	|	เป็นไฟล์หรือ Directory	|
|	system_conf_t	|	ชนิดของ object นี้ซึ่งตัวอย่างนี้คือ configuration type ของระบบ	|
|	s0	|	ระดับขั้นความสำคัญ	|

## Config file ของ SElinux
โดยปกติ SElinux จะมี Config file อยู่ที่
```bash
/etc/selinux/config
```
โดยจะมีบรรทัดที่สำคัญกำหนดเอาไว้คือ `SELINUX=?????` ตัวอย่างเช่น
```bash
SELINUX=enforcing
```
ซึ่งหมายถึง SElinux ถูกเปิดใช้งาน หากต้องการแก้ไขก็สามารถแก้ไขไฟล์ config นี้ให้เป็นค่าที่ต้องการได้ตามตารางที่ได้กล่าวถึงมาแล้ว

** หลังจากแก้ไขไฟล์ config แล้ว SElinux จะมีผลก็ต่อเมื่อทำการ reboot ระบบหลังจากแก้ไข config เท่านั้น หากไม่ reboot จะไม่มีผลใดๆ ระบบจะยังคงใช้ค่า config เดิมก่อนการแก้ไข

## การกำหนดค่า SElinux แบบชั่วคราว
หากไม่ต้องการ reboot เครื่องเพื่อเริ่มต้อนระบบใหม่ Linux อนุญาตให้เราทำการกำหนดค่า config ชั่วคราวของ SElinux ได้ด้วยคำสั่ง `setenforce` โดยมีรูปแบบดังนี้
```bash
setenforce <id>
```
โดยที่
|	ค่า id	|	คำอธิบาย	|
|:---|:---|
|	0	|	กำหนดให้ SElinux เป็น Permissive	|
|	1	|	กำหนดให้ SElinux เป็น Enforcing	|

** การกำหนดค่าชั่วคราวนี้จะมีผลทันทีหลังจากสั่งงาน ไม่จำเป็นต้อง reboot เริ่มต้นระบบใหม่ แต่หากมีการ reboot ค่าที่กำหนดชั่วคราวนี้จะถูก Override ด้วยค่าที่กำหนดเอาไว้ในไฟล์ config
