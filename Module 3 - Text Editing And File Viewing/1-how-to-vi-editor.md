# คู่มือการใช้งาน vi/vim

## `vi` คืออะไร

`vi` คือ text editor แบบ command-line ที่อยู่คู่กับระบบ Unix/Linux มานานมาก และบน RHEL มักจะมีให้ใช้งานอยู่แล้ว หรืออย่างน้อยก็มี editor ตระกูลเดียวกัน เช่น `vim` ที่ทำงานคล้ายกันมาก

**vi** นั้นเป็นโปรแกรมแก้ไขข้อความ (Text Editor) ที่เน้นการสั่งงานผ่านคีย์บอร์ดเป็นหลัก โดยมีปรัชญาว่า "มือของคุณไม่ควรออกจากแป้นพิมพ์" เพื่อประสิทธิภาพสูงสุดในการทำงาน

**vi** เป็นทักษะพื้นฐานที่คนดูแล Linux/RHEL "จำเป็นต้องรู้"

จุดเด่นของ `vi` คือ

- ใช้งานได้แม้ระบบมีแต่ terminal
- เบา เร็ว และเหมาะกับการแก้ไฟล์ config ในเครื่อง server
- ใช้ได้ผ่าน SSH โดยไม่ต้องมี GUI

จุดที่คนเริ่มต้นมักสับสนคือ `vi` **ไม่ได้พิมพ์ข้อความได้ทันทีเหมือน Notepad** เพราะมันมี “โหมด” การทำงาน

---

## 1. เข้าใจโหมดการทำงาน (3 Modes)
หัวใจสำคัญของ vi คือการที่มันมี **โหมด** การทำงานที่แยกออกจากกันชัดเจน แบ่งเป็น 3 โหมดหลักดังนี้:

1.  **Command Mode (Normal Mode):** เป็นโหมดเริ่มต้นเมื่อเปิดโปรแกรม ใช้สำหรับเลื่อนเคอร์เซอร์, ก๊อปปี้, วาง หรือลบข้อความ (พิมพ์ตัวอักษรลงไปจะไม่ปรากฏในไฟล์ แต่จะเป็นการสั่งงาน)
2.  **Insert Mode:** โหมดสำหรับพิมพ์ข้อความเหมือนโปรแกรมพิมพ์งานทั่วไป
3.  **Command-Line Mode (Ex Mode):** โหมดสำหรับสั่งงานระดับโปรแกรม เช่น บันทึกไฟล์, ออกจากโปรแกรม หรือการค้นหา/แทนที่


## 2. การเริ่มต้นและการออกจากโปรแกรม
* **เปิดไฟล์:** `vi filename.txt`
* **เข้าสู่ Insert Mode:** กด `i` (เพื่อพิมพ์ที่ตำแหน่งเคอร์เซอร์)
* **กลับสู่ Command Mode:** กดปุ่ม `Esc` (ใช้บ่อยที่สุด)
* **บันทึกและออก (Command-line Mode):**
    * `:w` — บันทึกไฟล์ (Write)
    * `:q` — ออกจากโปรแกรม (Quit)
    * `:wq` หรือ `:x` — บันทึกแล้วออก
    * `:q!` — ออกโดยไม่บันทึก (บังคับออก)


## 3. การเคลื่อนที่ (Navigation)
ใน Command Mode คุณสามารถใช้ลูกศรได้ แต่เซียน vi จะใช้ปุ่มเหล่านี้:
* `h` : ซ้าย
* `j` : ลง
* `k` : ขึ้น
* `l` : ขวา
* `0` : ไปต้นบรรทัด
* `$` : ไปท้ายบรรทัด
* `gg` : ไปบรรทัดแรกของไฟล์

- `^` = ไปตัวอักษรแรกของบรรทัด
- `$` = ไปท้ายบรรทัด
- `gg` = ไปต้นไฟล์
- `G` = ไปท้ายไฟล์
- `nG` = ไปบรรทัดที่ n เช่น `20G`
- `:20` = ไปบรรทัดที่ 20
- `v` = เลือกแบบตัวอักษร
- `V` = เลือกทั้งบรรทัด
- `Ctrl + f` = เลื่อนลงทีละหน้า
- `Ctrl + b` = เลื่อนขึ้นทีละหน้า
- `Ctrl + d` = ลงครึ่งหน้า
- `Ctrl + u` = ขึ้นครึ่งหน้า


**ตัวอย่าง:**
- `5j` = ลง 5 บรรทัด
- `10k` = ขึ้น 10 บรรทัด
- `3dd` = ลบ 3 บรรทัด
- `4yy` = คัดลอก 4 บรรทัด
- `20G` = ไปบรรทัด 20


## 4. การแก้ไขข้อความ (Editing)
ต้องทำใน **Command Mode** เท่านั้น:
* `x` : ลบตัวอักษรที่เคอร์เซอร์ทับ
* `dd` : ลบทั้งบรรทัด (ตัด)
* `dw` : ลบทั้งคำ (Delete Word)
* `yy` : ก๊อปปี้ทั้งบรรทัด (Yank)
* `p` : วาง (Paste) หลังจากบรรทัดปัจจุบัน
* `u` : Undo (ย้อนกลับการกระทำ)
* `Ctrl + r` : Redo (ทำซ้ำ)


## 5. การค้นหาและแทนที่ (Search and Replace)
* `/คำที่ต้องการค้นหา` : ค้นหาจากบนลงล่าง (กด `n` เพื่อไปคำถัดไป)
* `?คำที่ต้องการค้นหา` : ค้นหาจากล่างขึ้นบน
* `:%s/คำเก่า/คำใหม่/g` : เปลี่ยนคำเก่าเป็นคำใหม่ทั้งไฟล์


## 6. เคล็ดลับทางลัด (Pro Tips)
* **ทำซ้ำคำสั่ง:** หากต้องการลบ 5 บรรทัด ให้พิมพ์ `5dd`
* **ไปบรรทัดที่ระบุ:** พิมพ์ `:เลขบรรทัด` เช่น `:25` เพื่อไปบรรทัดที่ 25
* **เปิดเลขบรรทัด:** พิมพ์ `:set number`

## 7. ความแตกต่างระหว่าง `vi` กับ `vim`

### `vi`
- ดั้งเดิม
- ความสามารถพื้นฐาน
- เน้นใช้งานบนระบบ Unix แบบเก่าและเรียบง่าย

### `vim`
- ย่อมาจาก Vi IMproved
- มี feature มากกว่า
- มี syntax highlight, undo ดีกว่า, config ได้มากกว่า

**หมายเหตุ:**
`vi` แท้ดั้งเดิมอาจรองรับไม่เท่า `vim`
บน RHEL ส่วนใหญ่มักเจอ `vim` หรือ `vi` ที่ลิงก์ไปหา `vim`

## สรุปขั้นตอนการฝึกใช้งาน
1. เปิดไฟล์: `vi test.txt`
2. กด `i` เพื่อพิมพ์ข้อความ
3. พิมพ์เสร็จ กด `Esc`
4. พิมพ์ `:wq` เพื่อเซฟและออก
5. กด `Enter` เพื่อออกจาก vi

## ออกจาก vi โดยไม่บันทึกเมื่อแก้ผิด
1. กด `Esc`
2. พิมพ์ `:q!`
3. กด `Enter`

## LAB: How to use vi

### แบบฝึกหัด 1: สร้างไฟล์ข้อความ
```bash
vi practice.txt
```

ดำเนินการดังนี้:
- พิมพ์ 3 บรรทัด
- บันทึกไฟล์
- เปิดใหม่แล้วเพิ่มบรรทัดที่ 4

### แบบฝึกหัด 2: ลบ คัดลอก วาง
ในไฟล์เดิม:
- ลบบรรทัดที่ 2 ด้วย `dd`
- copy บรรทัดแรกด้วย `yy`
- paste ลงด้านล่างด้วย `p`

### แบบฝึกหัด 3: ค้นหาและแทนที่
สร้างคำว่า `server1` หลายจุด แล้วใช้

```text
:%s/server1/server2/gc
```

### แบบฝึกหัด 4: แก้ไฟล์ระบบจำลอง
```bash
cp /etc/hosts ~/hosts.lab
vi ~/hosts.lab
```

ฝึก:
- ค้นหา `localhost`
- เพิ่ม entry ใหม่
- บันทึก
- ตรวจผลด้วย `cat ~/hosts.lab`
