# 🧪 RHEL User & Group Administration Lab Exercises

คู่มือแบบฝึกหัดสำหรับการฝึกทักษะการบริหารจัดการผู้ใช้และกลุ่มบนระบบ RHEL (รวมถึง AlmaLinux,Rocky Linux, CentOS)

---

## 🛠 โจทย์ที่ 1: การสร้างโครงสร้างพื้นฐาน (Basic Onboarding)
**วัตถุประสงค์:** ฝึกการสร้างกลุ่มและการสร้างผู้ใช้พื้นฐานพร้อมกำหนดสิทธิ์เบื้องต้น

**สถานการณ์:** บริษัทเปิดแผนกใหม่ชื่อ **"DevOps"** และมีพนักงานใหม่ชื่อ **"Alice"** เข้ามาทำงาน
1. สร้างกลุ่มชื่อ `devops`
2. สร้าง User ชื่อ `alice` โดยให้มีเงื่อนไขดังนี้:
   - มี Comment ในระบบว่า "DevOps Engineer"
   - สังกัดกลุ่ม `devops` เป็นกลุ่มหลัก (Primary Group)
   - กำหนด Password เป็น `P@ssw0rd123`
3. ตรวจสอบข้อมูลของ `alice` ด้วยคำสั่ง `id` และดูในไฟล์ `/etc/passwd`

---

## 🛠 โจทย์ที่ 2: การจัดการสิทธิ์และการแชร์ไฟล์ (Collaboration)
**วัตถุประสงค์:** ฝึกการทำงานร่วมกันผ่านกลุ่ม (Group Collaboration) และการกำหนด Directory Permission

**สถานการณ์:** คุณต้องสร้างพื้นที่ส่วนกลางให้ทีม DevOps ทำงานร่วมกัน
1. สร้าง Directory ชื่อ `/opt/devops_data`
2. เปลี่ยนเจ้าของกลุ่ม (Group Owner) ของ Directory นี้ให้เป็นกลุ่ม `devops`
3. กำหนดสิทธิ์ (Permissions) โดยใช้ตัวเลข (Absolute Mode):
   - เจ้าของและสมาชิกกลุ่ม `devops` อ่าน/เขียน/เข้าถึงได้
   - บุคคลอื่น (Others) ไม่สามารถเข้าถึงโฟลเดอร์นี้ได้เลย
4. สร้าง User ชื่อ **"Bob"** ให้สังกัดกลุ่ม `devops` เป็น **กลุ่มเสริม (Supplementary Group)**
5. ทดสอบ Switch User เป็น `bob` แล้วลองสร้างไฟล์ใน `/opt/devops_data`

## 🛠 โจทย์ที่ 3: ความปลอดภัยและการจัดการ Account (Security & Policy)
**วัตถุประสงค์:** ฝึกการจัดการ Login Shell, การ Lock บัญชี และนโยบาย Password Aging

**สถานการณ์:** ปฏิบัติตามนโยบายความปลอดภัยเคร่งครัด
1. สร้าง User ชื่อ `app_service` สำหรับรันแอปพลิเคชัน โดย:
   - ห้ามมี Home Directory (`-M`)
   - ห้ามสามารถ Login เข้า Shell ได้ (ใช้ `/sbin/nologin`)
2. ปรับแต่ง User `alice` (จากโจทย์ที่ 1):
   - บังคับให้เปลี่ยนรหัสผ่านทันทีในการ Login ครั้งหน้า
   - กำหนดให้รหัสผ่านมีอายุใช้งานสูงสุด 30 วัน
3. ทำการ **Lock** บัญชีของ `bob` เพื่อระงับการใช้งานชั่วคราว และลองทดสอบ Login ด้วยชื่อ `bob`

---

## 🛠 โจทย์ที่ 4: การทำความสะอาดระบบ (System Cleanup)
**วัตถุประสงค์:** ฝึกการลบข้อมูลผู้ใช้ออกจากระบบอย่างสะอาดและปลอดภัย

**สถานการณ์:** พนักงานชื่อ `alice` ลาออกจากบริษัท
1. ลบ User `alice` ออกจากระบบ โดยต้องลบไฟล์ใน Home Directory ทิ้งทั้งหมด
2. ลบกลุ่ม `devops` ออกจากระบบ
3. ตรวจสอบไฟล์ `/etc/passwd` และ `/etc/group` เพื่อยืนยันว่าข้อมูลถูกลบจริง
