# RHEL ลดขนาด /home และเพิ่มขนาด / (os root)
ในการติดตั้ง OS ใหม่สำหรับ Linux ส่วนใหญ่แล้วค่า default ของขนาดพื้นที่ `/home` จะมีขนาดใหญ่มากเนื่องจากใน `/home` นั้นเก็บไฟล์และเก็บแต่ละ user ที่เข้ามาใช้งาน และในทางกลับกันขนาดของพื้นที่ `/` (os root) กลับมีขนาดที่เล็กและอาจจะเกิดปัญหาไม่เพียงพอในอนาคตขึ้นได้ เช่นปัญหา `/` เต็ม หรือ `/var` เต็ม ทำให้ OS ไม่สามารถรันต่อไปได้

ดังนั้นในการติดตั้งเราจึงควรพิจารณาให้ดีว่าเรากำหนดขนาดพื้นที่ของ `/ (os root)` เพียงพอกับการใช้งานหรือไม่ กำหนดขนาดของ `/var` เอาไว้เพียงพอต่อการเขียน log ของ Linux และ application ที่เราต้องการใช้งานหรือไม่

หากไม่ได้วางแผนเอาไว้และติดตั้งผ่านไปแล้ว แต่ไม่อยากทำการติดตั้งใหม่ เราจำเป็นต้องทำการลดขนาดของ `/home` หรือพาทอื่นๆ ที่ต้องการลดขนาดลงไป และทำการเพิ่มขนาดให้กับ `/` (os root) เพื่อให้ระบบรองรับตามที่เราต้องการ

ตัวอย่างนี้จะทำการลดขนาดของ /home เพื่อนำไปเพิ่มให้กับ `/` (os root) เนื่องจาก `xfs file system` ไม่มี `shrink function` จึงไม่สามารถลดขนาดเพิ่มขนาดได้โดยง่าย จึงจำเป็นต้องทำการ backup สิ่งที่จัดเก็บใน `/home` ออกก่อนและทำการลบ `/home` สร้าง `/home` ขึ้นใหม่ตามขนาดที่ต้องการ แล้วจึงเพิ่มขนาดของ `/` (os root) หลังจากนั้นจึงนำ backup เดิมของ `/home` มาทำการ restore กลับคืนข้อมูลเดิม

## ขั้นตอนการดำเนินการ
1. ตรวจสอบขนาดของพาทิชั่น
```bash
df -Th
```
2. Backup ข้อมูลของ `/home`
```bash
tar -cvpf /root/home_backup.tar /home
```
3. ตรวจสอบว่าใครกำลังใช้งานอยู่
ใช้คำสั่ง `lsof` หรือ `fuser `เพื่อดูว่ามี Process ID (PID) ไหนที่ขวางทางอยู่:
```bash
lsof +D /home
# หรือ
fuser -v -m /home
```
4. ตัดการเชื่อมต่อ
หากมั่นใจว่าไม่มีการเขียนข้อมูลสำคัญค้างอยู่ คุณสามารถสั่งปิด Process ทั้งหมดที่ใช้งาน `/home` ได้ทันทีด้วยคำสั่ง:
```bash
umount /home
# หรือ
fuser -km /home
```
> -k คือ kill process -m คือ moount path สามารถใช้รวมได้จึงเป็น -km <ชื่อ mount path>

ในกรณีที่ยัง unmount ไม่ได้ให้ลองใช้ Lazy Unmount คำสั่งคือ
```bash
umount -l /home
```
4. ลบ `/home` ออกจากระบบ
```bash
lvremove /dev/mapper/rhel-home
```
(ตอบ y เพื่อยืนยันการลบ)

5. สร้าง `/home` ขึ้นใหม่ตามขนาดที่ต้องการ
ในตัวอย่างนี้จะสร้าง `/home` ขึ้นใหม่ให้มีขนาดที่ `50G`
```bash
lvcreate -L 50G -n home rhel
mkfs.xfs /dev/mapper/rhel-home
mount /home
```
6. ขยายพื้นที่ให้ `/` (Root)
พื้นที่ว่างใน Volume Group เหลืออยู่ เราจะสั่งให้ `/` ขยายให้เต็มทั้งหมด
```bash
lvextend -l +100%FREE /dev/mapper/rhel-root
```
สั่งให้ File System ขยายตัวตามพื้นที่ที่เพิ่มขึ้นมา
```bash
xfs_growfs /
```
(ถ้าใช้ ext4 ให้ใช้คำสั่ง resize2fs /dev/mapper/rhel-root แทน)

7. Restore ข้อมูลของเกิมกลับมา
นำข้อมูลที่ Backup ไว้กลับมาที่เดิม
```bash
tar -xvpf /root/home_backup.tar -C /
```
8. ตรวจสอบขนาดของพาทิชั่นหลังแก้ไข
```bash
df -Th
```
### ข้อควรระวัง
ตรวจสอบไฟล์ `/etc/fstab` ปกติแล้วถ้าชื่อ Logical Volume เหมือนเดิม เราจะไม่ต้องแก้ไขไฟล์นี้ แต่ควรเช็คให้แน่ใจว่าเครื่องยังรู้จัก `/home` หลัง `Reboot` ต้องแน่ใจว่าชื่อ logical volume ต่า่งๆ ยังเป็นชื่อเดิม ไม่เช่นนั้นจะ `Reboot` ไม่ขึ้น

** หลังจากทำการขยายขนาด เราไม่จำเป็นต้อง `reboot` แต่หากต้องการแน่ใจว่าในการ `reboot` ครั้งต่อไปจะไม่เป็นปัญหา `reboot` ไม่ขึ้นให้ตรวจสอบให้แน่ใจและลองทำการ `reboot` หนึ่งครั้งเพื่อให้แน่ใจว่าระบบกลับมาทำงานได้จริง
